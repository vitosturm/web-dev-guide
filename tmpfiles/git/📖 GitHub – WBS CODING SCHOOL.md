### **Understanding GitHub: More than Git**

GitHub is a powerful tool for developers, often confused with Git. It's crucial to understand that GitHub is not Git. Git is a version control system that allows developers to manage changes to their codebase, whereas GitHub is a platform that hosts Git repositories online, providing a graphical interface and additional tools for collaborative development.

Yes, GitHub allows you to host your project’s code base in their server. Let’s you explore its files in a neat visual way, configure permissions for collaborators and secure your workflow, but it also bring some genuine new features on top of Git, chief among them **the Pull Request**

### Pull Requests on GitHub

A **Pull Request** (PR) on GitHub is a fundamental tool for collaboration in code projects. It is essentially a request made by a developer to merge their branch into another branch in the repository, typically the main project branch. The process begins once a developer pushes their branch to the repository and then proposes to integrate the changes.

Here's how it works:

-   **Proposing Changes:** After making changes in a separate branch, a developer creates a PR to propose these changes to the project's main branch.
-   **Review and Discussion:** Team members review the proposed changes, discuss potential improvements, and suggest modifications.
-   **Approval and Merge:** Once the PR is reviewed and approved by the necessary stakeholders, it can be merged, thereby integrating the changes into the main branch.

Pull Requests are vital for maintaining the integrity of the main project. They ensure that all changes are vetted and agreed upon before being added, facilitating a collaborative and error-minimizing environment.
