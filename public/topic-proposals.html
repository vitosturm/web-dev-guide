How Git Works — Step 1 / 4 topic-viz
📁
SCHRITT 01 — WORKING DIRECTORY
Hier beginnt alles
Das Working Directory ist dein normaler Projektordner. Git sieht zwar alle Dateien, trackt aber noch nichts. Jede Änderung passiert zuerst hier — ganz normal wie in jedem anderen Ordner.
$ ls index.html style.css app.js # unverändert — git weiß nichts davon
📦
SCHRITT 02 — STAGING AREA
Was kommt in den nächsten Commit?
Mit git add packst du Dateien in die Staging Area. Nur diese landen im Commit — du kannst also gezielt wählen welche Änderungen zusammengehören.
$ git add index.html $ git status Changes to be committed: modified: index.html
🗄️
SCHRITT 03 — LOCAL REPOSITORY
Ein Schnappschuss für die Ewigkeit
Der Commit speichert deinen Snapshot dauerhaft im lokalen Repository. Jeder Commit bekommt eine einzigartige ID — du kannst jederzeit zu einem alten Stand zurück.
$ git commit -m "fix: nav layout" [main abc123f] fix: nav layout 1 file changed, 3 insertions(+)
☁️
SCHRITT 04 — REMOTE (GITHUB)
Für alle sichtbar, sicher gesichert
Mit git push überträgst du deine Commits zu GitHub. Teamkollegen können jetzt git pull machen um deinen Stand zu bekommen. Backup inklusive.
$ git push origin main Enumerating objects: 5, done. ✓ main → origin/main
