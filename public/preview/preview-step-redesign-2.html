INFO 02

fetch() returns a Promise

The request is sent immediately but the response arrives later. fetch() returns a Promise that resolves when the server responds.

A fetch() call returns a Promise resolving to a Response when headers arrive. Call .json() to parse the body.

A fetch() call goes through two stages. First it resolves to a Response object when headers arrive. Then you call .json() or .text() to get another Promise that resolves when the full body is parsed.

The two-stage design lets you inspect headers and decide whether to stream the body — you can use response.body as a ReadableStream to process large files chunk by chunk.

const res = await fetch('/api/data')
// Stage 1: headers available
console.log(res.status) // 200
// Stage 2: read body
const data = await res.json()

fetch('/api/data') fires the HTTP request immediately and returns a pending Promise — your code does not block.

The Promise resolves to a Response object when the status + headers arrive. Body is not yet read — check res.ok here.

3
Body parsed → second await

Call await res.json() or res.text() to get a second Promise that resolves with the full body. Or stream via res.body.
