Option A Subtle Link — dezenter "Mehr erfahren"-Link unterm Text

Die kompakte Beschreibung bleibt im Vordergrund. Darunter ein kleiner monochromer Link — fast unsichtbar bis man ihn braucht. Klappt smooth auf und zeigt eine erweiterte Erklärung in neutralem Grauton.

How It Works
⏳

async/await syntax

async/await makes fetch code read like synchronous code — each await pauses the function until the Promise resolves, then continues with the result. Always wrap in try/catch because network failures reject the Promise.

Under the hood, async/await is identical to chaining .then() — it's purely syntactic sugar the JavaScript engine desugars before running. This means error propagation works the same way: a rejected Promise inside an await throws, which your catch block catches.

One important nuance: try/catch only catches network-level errors (no connection, DNS failure). A server responding with 404 or 500 still resolves the Promise — you must check response.ok yourself.

⚠️Multiple sequential awaits run one at a time. Use Promise.all([...]) to fire them in parallel.
⚠️Wrap in try/catch — network failures reject the Promise.
app.js
JS
async function loadUsers() {
  try {
    const res = await fetch("/api/users")
    const users = await res.json()
    return users
  } catch (err) {
    console.error("Request failed:", err)
  }
}

✅ Option A — Eigenschaften

  • Minimal-invasiv — der Link fällt kaum auf wenn man ihn nicht braucht
  • Text expandiert direkt im Card-Body (kein Popup/Modal)
  • Kein visueller Bruch im Layout
  • Einfachste Implementierung

Vergleich

Alle Optionen bieten On-Demand-Details ohne den Basis-Inhalt zu überladen.

A — Subtle Link Unsichtbarster Trigger

Kleiner "+ Mehr erfahren"-Link unter dem Text. Kaum sichtbar — stört den Lesefluss nicht. Expandiert inline im Card-Body. Empfehlung für minimale UI.

B — Dashed Border Klares visuelles Signal

Gestrichelte Box als erkennbare "optionale Zone". Icon + Label kommunizieren den Inhalt ohne Klick. Beste Affordance — Nutzer verstehen sofort was sie erwartet.

C — Inline Accordion Teil der Karte

Accordion am unteren Card-Rand. Eigener Hintergrund trennt den Detail-Bereich visuell. Strukturierteste Lösung — Kern-Inhalt und Details klar getrennt.

D — Floating Drawer Trennung von Step und Detail

Details-Button im Header, Panel erscheint unter der Karte. Step bleibt komplett unverändert. Sauberste Trennung — der Step-Inhalt wird nie verändert.

⚡ Empfehlung

Option B (Dashed Border) oder Option C (Inline Accordion) — beide machen den optionalen Charakter sofort erkennbar ohne den Haupt-Inhalt zu überladen. Option C hat die klarste Struktur, Option B die stärkste visuelle Einladung.
