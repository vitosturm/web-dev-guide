INFO 02
Array Methods

Arrays have built-in methods for transforming and querying data.

Method
What it does
.map()
Returns a new array — transforms every item
.filter()
Returns a new array — keeps only matching items
.reduce()
Reduces all items to a single value
.find()
Returns the first matching item (or undefined)
.includes()
Returns true / false — checks if value exists
arrays.js
JS
const nums = [1, 2, 3, 4]
nums.filter(n => n > 2)  // [3, 4]
nums.map(n => n * 2)    // [2, 4, 6, 8]

INFO 01
const vs let

Use const by default. Only use let when you need to reassign.

Avoid
let name = 'Alice'
let PI = 3.14
let config = {}
Prefer
const name = 'Alice'
const PI = 3.14
const config = {}
⚡

const prevents accidental reassignment — the linter will catch the mistake before it runs.

🎨
INFO 02
Specificity — right vs wrong selector

Overly specific selectors break re-use and create debugging headaches.

Too specific
header nav ul li a {
  color: blue;
}
Targeted
.nav-link {
  color: blue;
}
