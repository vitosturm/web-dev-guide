js-variables — Step 1: var — the old way Farbe: #fbbf24 (JS)
VorherJetzt
INFO 01 var — the old way

var is function-scoped and hoisted, which causes unexpected bugs. Modern JavaScript avoids it.

var

var is function-scoped: it leaks out of if blocks and for loops into the nearest enclosing function. The engine also hoists var declarations to the top of the function, initialising them to undefined.

let & const

let and const are block-scoped. Reading them before their declaration throws a ReferenceError. This is the "temporal dead zone" — the period between entering the block and the declaration line.

css-flexbox — Step 1: The classic navbar Farbe: #38bdf8 (CSS)
VorherJetzt
INFO 01 The classic navbar

Flexbox solves the navbar in 2 lines: display: flex on the container, then justify-content: space-between pushes the logo left and links right.

Flexbox

Flexbox replaced older techniques like float-based navbars. The two-line pattern display: flex; justify-content: space-between achieves what previously required calculating widths, adding clearfix hacks, and fighting mysterious whitespace issues.

Flex formatting

The flex container establishes a flex formatting context. Direct children become flex items and gain access to flex properties (flex-grow, flex-shrink, etc.).

Warum Flexbox?

Flexbox löste float-basierte Navbars ab — was früher berechnete Breiten, Clearfix-Hacks und mysteriöse Whitespace-Probleme brauchte, geht jetzt in zwei Zeilen.

Merkhilfe

Hauptachse vs. Querachse: justify-content steuert die Hauptachse, align-items die Querachse. Mit flex-direction: column drehen sich beide.

react-state — Step 1: State is dynamic data Farbe: #61dafb (React)
VorherJetzt
INFO 01 State is dynamic data

State is data that can change over time and causes the component to re-render when it does. Props come from outside; state lives inside.

useState

useState is the fundamental hook for managing local component state. Without it, variables declared inside a function component reset to their initial value on every render.

Re-render

Every time you call the setter, React schedules a re-render. State updates are asynchronous — the new value is not immediately available in the same event handler call.

🔄
INFO 01 State is dynamic data

State is data that can change over time and causes the component to re-render when it does. Props come from outside; state lives inside.

🔬
Wie useState funktioniert

useState speichert den Wert in einem React-internen Slot, der zwischen Renders erhalten bleibt — normale Variablen im Funktionskörper werden bei jedem Render neu initialisiert.

⚠️
State-Updates sind asynchron

Nach dem Setter-Aufruf ist der neue Wert nicht sofort verfügbar — React schedulet einen Re-render. Nutze den funktionalen Update setCount(c => c + 1) wenn du auf den vorherigen Wert angewiesen bist.

✅
Wann State, wann Props?

Faustregel: Kann der Wert von außen gesteuert werden? → Props. Gehört der Wert nur diesem Component? → State. State nach oben zu heben ist immer möglich; nach unten schieben kaum.
