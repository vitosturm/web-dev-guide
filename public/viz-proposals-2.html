Git Workflow — Glassmorphism
🔄
STEP 01 · INITIALIZE
Repository starten
git init erzeugt den .git-Ordner und startet die Versionsverfolgung im Projekt.
git init
📥
STEP 02 · STAGE
Änderungen stagen
git add überträgt geänderte Dateien in die Staging-Area — bereit für den nächsten Commit.
git add .
🌿
STEP 03 · COMMIT
Snapshot speichern
git commit legt einen permanenten Snapshot im Verlauf ab — mit Botschaft und Zeitstempel.
git commit -m "feat: auth"
📦
STEP 04 · BRANCH
Branch erstellen
git branch öffnet eine parallele Zeitlinie — Experimente ohne main zu berühren.
git checkout -b feature/login
🚀
STEP 05 · PUSH
Remote synchronisieren
git push synchronisiert deinen Branch mit dem Remote-Repository auf GitHub.
git push origin feature/login
